# Control Plane Demo Flow

This guide walks through a live demo of the Nzila OS Control Plane — the executive
and operator entry point for platform health, governance, AI intelligence,
anomaly detection, agent workflows, and procurement evidence.

## Prerequisites

```bash
pnpm install
pnpm demo:seed           # optional: seed demo data
pnpm --filter @nzila/control-plane dev
```

Open **http://localhost:3010** — you'll be redirected to `/overview`.

## Demo Script

### 1. Overview Dashboard (2 min)

- **Platform Health**: shows aggregate health (healthy / degraded / critical).
- **Governance Summary**: overall status, score, last audit timestamp.
- **Intelligence Summary**: cross-app insight count and last generated time.
- **Module Status**: counts modules by health state.
- **Procurement Summary**: latest pack ID, signature and attestation status.
- **Recent Anomalies**: top-3 anomalies sorted by severity.

### 2. Governance Page (2 min)

Navigate to **Governance** in the sidebar.

- Review the status cards: overall status, compliance score, pending actions.
- Scroll down to the procurement pack attestation section.
- Review the audit timeline — each entry shows who changed what and when.

### 3. Intelligence Page (2 min)

Navigate to **Intelligence**.

- Review the cross-app insights generated by the intelligence layer.
- Browse operational signals with severity and source app.
- Use the **Query Box** to ask a natural-language question:
  - Example: _"What is the compliance status of the trade module?"_
  - The response shows confidence, sources, and evidence references.

### 4. Anomalies Page (2 min)

Navigate to **Anomalies**.

- Anomalies are sorted by severity (critical → low).
- Each card shows: type, severity, app, metric, expected/actual values, deviation factor.
- The "Recommended Next Step" section provides actionable guidance.

### 5. Agent Workflows (2 min)

Navigate to **Agents**.

- Recommendations are sorted by priority.
- Look for the **"Human review required"** badge on critical recommendations.
- Each recommendation shows category, confidence, source, and evidence references.
- **No mutation actions are exposed** — this is a read-only recommendation view.

### 6. Modules Page (1 min)

Navigate to **Modules**.

- Grid view of all registered platform modules.
- Each card shows version, status, health, and governance integration.

### 7. Procurement Page (2 min)

Navigate to **Procurement**.

- Review the evidence pack: Pack ID, manifest hash, SBOM reference.
- Verify the signature status (verified) and attestation status (valid).
- Review the 5 supply-chain evidence sections: Security, Data Lifecycle,
  Operational, Governance, Sovereignty.

## API Endpoints

All data is available via REST:

| Endpoint | Method | Description |
|---|---|---|
| `/api/control-plane/governance/status` | GET | Governance status |
| `/api/control-plane/governance/timeline` | GET | Audit timeline |
| `/api/control-plane/intelligence/summary` | GET | Cross-app insights |
| `/api/control-plane/intelligence/query` | POST | Natural-language query |
| `/api/control-plane/intelligence/signals` | GET | Operational signals |
| `/api/control-plane/anomalies` | GET | All anomalies |
| `/api/control-plane/anomalies/:id` | GET | Single anomaly |
| `/api/control-plane/agents/recommendations` | GET | Agent recommendations |
| `/api/control-plane/modules` | GET | Module registry |
| `/api/control-plane/procurement/latest` | GET | Latest evidence pack |
| `/api/control-plane/procurement/public-key` | GET | Pack signing key |
| `/api/control-plane/procurement/validate` | POST | Validate pack signature |
